Maptool Basics: Starting a Server

This is what I do to start a server. I should warn you that if it doesn’t work for you, there’s not a lot I can do to help – this is the part of Maptool I understand the least. You may find it useful to consult the Networking Help guide that the guys at RPTools wrote, but it isn’t super obvious to me.

Still, Rich, Scott, and I have all managed it, so hopefully it’s doable for you, too.

To start a server, go to the File menu, and click "Start a Server." You'll get a dialogue that looks something like this:

Some of this is pretty opaque to me, but:

  • Username is the name you log in as.
  • You can log in as a GM or as a Player to your own server. You will not need a password for either, because that would be silly.
  • Port is the server port. The default for Maptool is 51234, so leave that alone.
  • UPnP is a handy feature that your router might have. Mine is old and does. Try checking this. If you don’t have UPnP, things get difficult.

Obviously the RPTools.net Alias is the name the server shows up as on the list, GM Password is the password for other people to use if they want to log in as a GM, and Player Password is the password for players to log in with.

Everything in the Options box you should set to be exactly identical to what I have done.

Sometimes people can’t see the server in the list, so we have to make a direct connection (what Fey calls the back door). You’ll need to provide people with the IP address of your server. The easiest way to do that is to go to the File menu and pick "Connection Information." You'll get a little dialogue like so.

Write down the 10-digit number that appears under External Address and email it to people. I’ve put a little black box over mine out of paranoia, since this is a public website.
